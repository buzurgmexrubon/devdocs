# O'rnatish

Docusaurus bir qator npm [paketlaridan](https://github.com/facebook/docusaurus/tree/main/packages) iborat.

:::tip

Docusaurus'ni darhol brauzeringizda sinab ko'rish uchun **[docusaurus.new](https://docusaurus.new)** dan foydalaning!

:::

## Talablar

- [Node.js](https://nodejs.org/en/download/) 18.0 yoki undan yuqori versiyasi (`node -v` buyrug'ini ishga tushirish orqali tekshirish mumkin). Bitta mashinada bir nechta Node.js versiyalarini boshqarish uchun [nvm](https://github.com/nvm-sh/nvm) dan foydalanishingiz mumkin.
  - Node.js'ni o'rnatayotganda, barcha bog'liqliklarga (dependencies) oid katakchalarni belgilash tavsiya etiladi.

## Scaffold project website

The easiest way to install Docusaurus is to use the [`create-docusaurus`](./api/misc/create-docusaurus.mdx) command line tool that helps you scaffold a skeleton Docusaurus website. You can run this command anywhere in a new empty repository or within an existing repository, it will create a new directory containing the scaffolded files.

```bash
npx create-docusaurus@latest my-website classic
```

We recommend the `classic` template so that you can get started quickly, and it contains features found in Docusaurus 1. The `classic` template contains `@docusaurus/preset-classic` which includes standard documentation, a blog, custom pages, and a CSS framework (with dark mode support). You can get up and running extremely quickly with the classic template and customize things later on when you have gained more familiarity with Docusaurus.

You can also use the template's TypeScript variant by passing the `--typescript` flag. See [TypeScript support](./typescript-support.mdx) for more information.

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info Meta-Only

If you are setting up a new Docusaurus website for a Meta open source project, run this command inside an internal repository, which comes with some useful Meta-specific defaults:

```bash
scarf static-docs-bootstrap
```

:::

<details>
  <summary>Alternative installation commands</summary>

You can also initialize a new project using your preferred project manager:

```bash npm2yarn
npm init docusaurus
```

</details>

Run `npx create-docusaurus@latest --help`, or check out its [API docs](./api/misc/create-docusaurus.mdx) for more information about all available flags.

## Loyiha veb-saytining asosini yaratish (Scaffold project website)

Docusaurusni o'rnatishning eng oson yo'li bu [`create-docusaurus`](./api/misc/create-docusaurus.mdx) buyruq qatori (command line) vositasidan foydalanishdir, u sizga Docusaurus veb-saytining skeletini yaratishga yordam beradi. Siz bu buyruqni yangi bo'sh repozitoriyda yoki mavjud repozitoriy ichida istalgan joyda ishga tushirishingiz mumkin, u yaratilgan fayllarni o'z ichiga olgan yangi papka hosil qiladi.

```bash
npx create-docusaurus@latest my-website classic
```

Tezroq ishga kirishishingiz uchun biz `classic` shablonini tavsiya etamiz. Chunki u Docusaurus 1 dagi mavjud funksiyalarni o'z ichiga oladi. "Classic" shabloni `@docusaurus/preset-classic` ni o'zida mujassam etgan bo'lib, unga standart dokumentatsiya, blog, shaxsiy sahifalar va CSS freymvorki (qorong'u rejimni qo'llab-quvvatlaydi) kiradi. "Classic" shabloni yordamida loyihangizni juda tez ishga tushirishingiz va Docusaurus bilan yaxshiroq tanishganingizdan so'ng uni keyinchalik o'zgartirishingiz mumkin bo'ladi.

Shablonning TypeScript variantini `--typescript` flagini berish orqali ham ishlatishingiz mumkin. Qo'shimcha ma'lumot uchun [TypeScript-ni qo'llab-quvvatlash](./typescript-support.mdx) bo'limiga qarang.

```bash
npx create-docusaurus@latest my-website classic --typescript
```

:::info Faqat Meta-uchun

Agar siz Meta ochiq kodli loyihasi uchun yangi Docusaurus veb-saytini o'rnatayotgan bo'lsangiz, ushbu buyruqni ichki (internal) repozitoriyda ishga tushiring. Bu buyruq Meta'ga xos ba'zi foydali standart sozlamalar bilan birga keladi.

```bash
scarf static-docs-bootstrap
```

:::

### Loyiha Tuzilmasiga Qisqacha Sharh

- **`/blog/`** - Bu papkada **blog yozuvlari uchun Markdown fayllari** joylashadi. Agar blog plaginini oʻchirib qoʻygan boʻlsangiz, bu papkani oʻchirib tashlashingiz yoki `path` parametrini oʻzgartirishingiz mumkin. Batafsil maʼlumotni [blog qoʻllanmasida](../guides/blog.mdx) topishingiz mumkin.
  - **`authors.yml`**: Blog yozuvlari mualliflari haqidagi ma'lumotlarni saqlaydi.
  - **`tags.yml`**: Blog postlarining teglarini (kalit so'zlarini) belgilaydi.
- **`/docs/`** - Dokumentatsiya uchun **Markdown fayllari** shu yerda joylashgan. `sidebars.js` faylida dokumentatsiya yon panelining tartibini moslashtirishingiz mumkin. Agar docs plaginini oʻchirib qoʻygan boʻlsangiz, bu papkani oʻchirib tashlashingiz yoki `path` parametrini oʻzgartirishingiz mumkin. Batafsil maʼlumotni [docs qoʻllanmasida](../guides/docs/) topishingiz mumkin.
  - **`tutorial-basics/` va `tutorial-extras/`**: Tutorial (qo'llanma) bo'limlari uchun alohida papkalar.
  - **`_category_.json`**: Docs yon panelidagi kategoriyalarning nomini va tartibini sozlash uchun ishlatiladi.
  - **`img/`**: Dokumentatsiya ichida ishlatiladigan rasmlar uchun.
- **`/src/`** - Sahifalar yoki maxsus React komponentlari kabi **dokumentatsiyaga kirmaydigan fayllar** joylashadigan joy. Bu fayllarni qat'iy shu yerga joylashtirishingiz shart emas, ammo ularni bir markazlashgan papkaga qo'yish, agar sizga qandaydir **linting/processing (ishlov berish)** kerak bo'lsa, uni osonlashtiradi.
  - **`/src/components`**: Maxsus React komponentlaringizni joylashtirish uchun.
  - **`/src/css`**: Saytingizning umumiy CSS fayllari, masalan, `custom.css` joylashadi.
  - **`/src/pages`**: Bu papka ichidagi har qanday **JSX/TSX/MDX fayli** veb-sayt sahifasiga aylanadi. Batafsil maʼlumotni [sahifalar qoʻllanmasida](../guides//creating-pages.mdx) topishingiz mumkin.
- **`/static/`** - Bu papka ichidagi barcha fayllar va papkalar yakuniy build papkasining ildiziga nusxalanadi. Bu yerda sizning rasmlaringiz, favikoningiz va boshqa statik resurslaringiz saqlanadi.
- **`docusaurus.config.js`** - Sayt konfiguratsiyasini oʻz ichiga olgan konfiguratsiya fayli. Bu Docusaurus v1 dagi `siteConfig.js` fayliga teng.
- **`package.json`** - **Docusaurus veb-sayti React ilovasi hisoblanadi.** Siz unda istalgan npm paketlarini oʻrnatishingiz va ishlatishingiz mumkin.
- **`package-lock.json`** - npm paket bog'liqliklarining aniq versiyalarini qulflash uchun `package.json` bilan birga ishlatiladi.
- **`README.md`** - Loyihangiz haqida umumiy ma'lumot beruvchi fayl.
- **`sidebars.js`** - Dokumentatsiyaning yon panelidagi sahifalar tartibini belgilash uchun ishlatiladi.

### Monorepolar

Agar siz mavjud loyihaning dokumentatsiyasi uchun Docusaurusdan foydalanayotgan bo'lsangiz, monorepo siz uchun yechim bo'lishi mumkin. Monorepolar o'xshash loyihalar o'rtasida bog'liqliklarni (dependencies) almashish imkonini beradi. Misol uchun, veb-saytingiz yangi funksiyalarni ko'rsatish uchun chiqarilgan versiyaga bog'liq bo'lmasdan, mahalliy paketlaringizdan foydalanishi mumkin. Keyin, kontributorlar funksiyalarni amalga oshirayotganda hujjatlarni yangilashlari mumkin. Quyida monorepo papka tuzilishi misoli keltirilgan:

```bash
my-monorepo
├── package-a # Boshqa paket, sizning haqiqiy loyihangiz
│   ├── src
│   └── package.json # A paketi bog'liqliklari
├── website   # Docusaurus ildizi
│   ├── docs
│   ├── src
│   └── package.json # Docusaurus bog'liqliklari
├── package.json # Monorepo umumiy bog'liqliklari
```

Bu holda, siz `npx create-docusaurus` buyrug'ini `./my-monorepo` papkasida ishga tushirishingiz kerak.

Agar siz Netlify yoki Vercel kabi hosting provayderidan foydalanayotgan bo'lsangiz, saytning `Base directory` (asosiy papka) sini Docusaurus loyihangiz joylashgan ildiz papkasiga o'zgartirishingiz kerak bo'ladi. Bu holatda, u ./website bo'ladi. Konfiguratsiyalar haqida batafsil ma'lumotni [joylashtirish (deployment) dokumentatsiyasida](../guides/deployment.mdx#netlify-ga-joylashtirish-deploy) o'qishingiz mumkin.

Monorepolar haqida ko'proq ma'lumotni [Yarn dokumentatsiyasida](https://yarnpkg.com/features/workspaces) o'qing (Yarn monorepo o'rnatishning yagona yo'li emas, lekin u keng tarqalgan yechim) yoki real hayotiy misollar uchun [Docusaurus](https://github.com/facebook/docusaurus) va [Jest](https://github.com/facebook/jest) ni ko'rib chiqing.

## Updating your Docusaurus version

There are many ways to update your Docusaurus version. One guaranteed way is to manually change the version number in `package.json` to the desired version. Note that all `@docusaurus/`-namespaced packages should be using the same version.

<UpgradeGuide />

Then, in the directory containing `package.json`, run your package manager's install command:

<Tabs groupId="package-installation">
  <TabItem value="npm" label="npm">
    ```bash
    npm install
    ```
  </TabItem>
  <TabItem value="yarn" label="Yarn">
    ```bash
    yarn install
    ```
  </TabItem>
  <TabItem value="pnpm" label="pnpm">
    ```bash
    pnpm install
    ```
  </TabItem>
  <TabItem value="bun" label="Bun">
    ```bash
    bun install
    ```
  </TabItem>

</Tabs>

:::tip

`npm install` may report several vulnerabilities and recommend running `npm audit` to address them. Typically, these reported vulnerabilities, such as RegExp DOS vulnerabilities, are harmless and can be safely ignored. Also read this article, which reflects our thinking: [npm audit: Broken by Design](https://overreacted.io/npm-audit-broken-by-design/).

:::

To check that the update occurred successfully, run:

```bash
npx docusaurus --version
```

You should see the correct version as output.

Alternatively, if you are using Yarn, you can do:

```bash
yarn add @docusaurus/core @docusaurus/preset-classic
```

:::tip

Use new unreleased features of Docusaurus with the [`@canary` npm dist tag](/community/canary)

:::

## Docusaurus versiyasini yangilash

Docusaurus versiyasini yangilashning ko'plab usullari mavjud. Kafolatlangan usullardan biri `package.json` faylidagi versiya raqamini qo'lda kerakli versiyaga o'zgartirishdir. Shuni esda tutingki, barcha `@docusaurus/`-nomli paketlar bir xil versiyadan foydalanishi kerak.

  </TabItem>
  <TabItem value="pnpm" label="pnpm">

    ```bash
    pnpm install
    ```
  </TabItem>
  <TabItem value="bun" label="Bun">

:::tip

`npm install` bir nechta zaifliklarni xabar qilishi va ularni bartaraf etish uchun `npm audit`ni ishga tushirishni tavsiya qilishi mumkin. Odatda, bu xabar qilingan zaifliklar, masalan, RegExp DOS zaifliklari, zararsizdir va ularni xavfsiz tarzda e'tiborsiz qoldirish mumkin. Shuningdek, bizning fikrimizni aks ettiruvchi ushbu maqolani o'qing: [npm audit: Broken by Design](https://overreacted.io/npm-audit-broken-by-design/).

:::

Yangilanish muvaffaqiyatli amalga oshirilganligini tekshirish uchun quyidagilarni ishga tushiring:

```bash
npx docusaurus --version
```

Siz to'g'ri versiyani chiqish sifatida ko'rishingiz kerak.

Muqobil ravishda, agar siz Yarn dan foydalanayotgan bo'lsangiz, quyidagilarni qilishingiz mumkin:

```bash
yarn add @docusaurus/core @docusaurus/preset-classic
```

:::tip

Docusaurusning yangi chiqarilmagan xususiyatlaridan [`@canary` npm dist tagi](https://www.google.com/search?q=/community/canary) bilan foydalaning

:::
