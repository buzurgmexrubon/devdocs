## Tezkor Yo'l ‚è±Ô∏è

Docusaurus'ni **5 daqiqada** o'rganing!

Yangi Docusaurus saytini yarating va ichiga o'rnatilgan **juda qisqa** qo'llanmaga amal qiling.

[Node.js](https://nodejs.org/en/download/) ni o'rnating va yangi Docusaurus saytini yarating:

```bash
npx create-docusaurus@latest my-website classic
```

Saytni ishga tushiring:

```bash
cd my-website
npx docusaurus start
```

[`http://localhost:3000`](http://localhost:3000) ni oching va qo'llanmaga amal qiling.

:::tip

Docusaurus'ni darhol brauzeringizda sinab ko'rish uchun **[docusaurus.new](https://docusaurus.new)** dan foydalaning\!

:::

## v1 dan migratsiya qilish

Docusaurus v2+ butunlay modernizatsiya qilingan asboblar to'plamidan foydalangan holda Docusaurus v1 dan qayta yozilgan. v2 ning rasmiy nashridan so'ng, Docusaurus v1 eskirganligi sababli sizni Docusaurus v1 o'rniga **Docusaurus v2+ dan foydalanishga** qattiq chorlaymiz.

**Docusaurus v2+ dan foydalaning, agar:**

- :white_check_mark: Siz zamonaviy Jamstack dokumentatsiyalar saytini xohlasangiz
- :white_check_mark: Siz mijoz tomoni marshrutlash (client-side routing) bilan bir sahifali ilova (SPA) xohlasangiz
- :white_check_mark: Siz React va MDX ning butun kuchidan foydalanmoqchi bo'lsangiz
- :white_check_mark: Sizga IE11 uchun yordam kerak bo'lmasa

**Docusaurus v1 dan foydalaning, agar:**

- :x: Siz bir sahifali ilova (SPA) xohlamasangiz
- :x: Sizga IE11 uchun yordam kerak bo'lsa (IE [allaqachon xizmat muddati tugagan](https://docs.microsoft.com/en-us/lifecycle/products/internet-explorer-11) va endi rasman qo'llab-quvvatlanmaydi)

V1 dan v2+ ga yangilamoqchi bo'lsangiz, [migratsiya qo'llanmalarimizga](./migration/index.mdx) amal qilishingiz mumkin.

## Imkoniyatlar (Features)

Docusaurus dasturchi va kontributor tajribasiga katta e'tibor qaratib yaratilgan.

- ‚öõÔ∏è **üíö va React bilan yaratilgan**:
  - React yordamida kengaytirish va sozlash
  - O'z React komponentlaringizni taqdim etish orqali saytingizning ko'rish tajribasini (browsing experience) to'liq nazorat qiling
- üîå **Plaginlar bilan ishlaydi (Pluggable)**:
  - Saytingizni asosiy shablon bilan ishga tushiring, so'ngra ilg'or funksiyalar va plaginlardan foydalaning
  - Plaginlaringizni jamoa bilan bo'lishish uchun ochiq kodli qiling
- ‚úÇÔ∏è **Dasturchi tajribasi (Developer experience)**:
  - Hujjatlaringizni hoziroq yozishni boshlang
  - Universal konfiguratsiya kirish nuqtasi uni hissa qo'shuvchilar tomonidan yanada qulayroq qilish uchun
  - O'zgarishlar bo'yicha yashin tezligida qayta yuklash
  - Marshrutga asoslangan kod va ma'lumotlarni ajratish
  - GitHub Pages, Netlify, Vercel va boshqa joylashtirish xizmatlariga osonlik bilan nashr (publish) qiling

Bizning umumiy maqsadimiz ‚Äî foydalanuvchilaringizga kerakli narsani tezda topishga va mahsulotlaringizni yaxshiroq tushunishga yordam berish. Biz dokumentatsiya saytlaringizni to'g'ri va yaxshi yaratishingizga yordam berish uchun eng yaxshi amaliyotlarimizni baham ko'ramiz.

- üéØ **SEO uchun qulay (SEO friendly)**:
  - HTML fayllari har bir mumkin bo'lgan yo'l uchun statik ravishda yaratiladi.
  - Foydalanuvchilaringizga ularning muammolari bilan bevosita bog'liq bo'lgan rasmiy dokumentatsiyalaringizga kirishga yordam berish uchun sahifaga xos SEO.
- üìù **MDX ustiga qurilgan**:
  - Markdown ichiga joylashtirilgan JSX va React orqali interaktiv komponentlar yozing.
  - Foydalanuvchilaringiz mahsulotlaringizni shu zahotiyoq sevishi uchun kodlaringizni jonli muharrirlarda baham ko'ring.
- üîç **Qidiruv**: Sizning to'liq saytingizni qidirish mumkin.
- üíæ **Hujjat versiyalash (Document Versioning)**: Hujjatlarni loyiha relizlari bilan sinxron saqlashga yordam beradi.
- üåç **Xalqaro (i18n)**: Saytingizni bir nechta tillarga tarjima qiling.

Docusaurus v2+ barcha foydalanuvchilaringiz uchun qulay va chaqmoq tezligida foydalanish uchun yaratilgan.

- ‚ö°Ô∏è **Chaqmoq tezligida**. Docusaurus v2+ kontentingizning juda tez yuklanishini ta'minlaydigan [PRPL Pattern](https://developers.google.com/web/fundamentals/performance/prpl-pattern/) ga amal qiladi.
- ü¶ñ **Qulay (Accessible)**. Saytingiz barcha foydalanuvchilar uchun bir xil darajada qulay bo'lishiga e'tibor qaratilgan.

## Dizayn prinsiplari

- **Kam o'rganish kerak**. Docusaurusni o'rganish va undan foydalanish oson bo'lishi kerak, chunki API juda kichik. Ko'pgina narsalarni foydalanuvchilar o'zlari amalga oshira oladilar, hatto buning uchun ko'proq kod va vaqt talab qilsa ham. Noto'g'ri abstraksiyalarga ega bo'lishdan ko'ra, abstraksiyalarsiz bo'lish yaxshiroq va biz foydalanuvchilarning noto'g'ri abstraksiyalar atrofida ishlashlariga to'g'ri kelishini istamaymiz. Zaruriy ma'ruza ‚Äî [Minimal API Surface Area](https://www.youtube.com/watch?v=4anAwXYqLG8).
- **Intuitiv**. Foydalanuvchilar Docusaurus loyihasining papkasini ko'rganlarida yoki yangi funksiyalarni qo'shganlarida qiynalmaydilar. Ularga tanish bo'lgan yondashuvlardan foydalangan holda, u intuitiv ko'rinishi va yaratish oson bo'lishi kerak.
- **Qatlamli arxitektura**. Bizning stekimizning har bir qatlami (kontent/mavzu/stil) o'rtasidagi masalalar ajratilishi aniq bo'lishi kerak ‚Äî yaxshi abstraksiyalangan va modulli.
- **Oqilona standart sozlamalar**. Umumiy va mashhur ishlash optimizatsiyalari hamda konfiguratsiyalar foydalanuvchilar uchun amalga oshiriladi, ammo ularga ularni bekor qilish (o'zgartirish) imkoniyati beriladi.
- **Yetkazib beruvchiga bog'lanib qolish yo'q (No vendor lock-in)**. Foydalanuvchilar standart plaginlar yoki CSSdan foydalanishga majbur emaslar, garchi bu qattiq tavsiya etilsada. React Loadable va React Router kabi ayrim asosiy infratuzilmalar almashtirib bo'lmaydi, chunki biz ularda standart ishlash optimizatsiyasini amalga oshiramiz, biroq yuqori darajadagilarida emas. Markdown dvigatellari, CSS freymvorklari, CSS metodologiyasi va boshqa arxitekturalarni tanlash butunlay foydalanuvchilarning ixtiyorida bo'ladi.

Biz dasturchilar sifatida kutubxonaning qanday ishlashini bilish undan yaxshiroq foydalanishga yordam beradi deb hisoblaymiz. Shuning uchun biz Docusaurus arxitekturasi va turli komponentlarini tushuntirishga harakat qilamiz, toki uni o'qigan foydalanuvchilar vosita (tool)ni chuqurroq tushunishadi va undan yanada samaraliroq foydalana oladilar.

## Boshqa vositalar bilan taqqoslash

Barcha statik sayt generatorlari orasida Docusaurus dokumentatsiya saytlariga alohida e'tibor qaratadi va ko'plab tayyor funksiyalarga ega.

Biz boshqa asosiy statik sayt generatorlarini ham o'rganib chiqdik va taqqoslash bo'yicha o'z tushunchalarimizni bo'lishmoqchimiz, umid qilamizki, bu sizga mavjud bo'lgan xilma-xil tanlovlar orasida yo'l topishga yordam beradi.

### Gatsby

[Gatsby](https://www.gatsbyjs.com/) ko'plab imkoniyatlarga boy, boy plagin ekotizimiga ega va Docusaurus qiladigan barcha ishlarni qila oladi. Tabiiyki, bu yuqoriroq o'rganish "egri chizig'iga" (learning curve) olib keladi. Gatsby ko'p ishlarni yaxshi bajaradi va ko'plab turdagi veb-saytlarni yaratish uchun javob beradi. Boshqa tomondan, Docusaurus bitta ishni juda yaxshi bajarishga harakat qiladi - kontent yozish va nashr qilish uchun eng yaxshi vosita bo'lish.

GraphQL ham Gatsby uchun juda muhim, garchi Gatsby saytini yaratish uchun GraphQL shart bo'lmasa ham. Statik veb-saytlarni yaratishda ko'pgina hollarda sizga GraphQL taqdim etadigan moslashuvchanlik kerak bo'lmaydi.

Docusaurus v2+ ning ko'plab jihatlari Gatsby'ning eng yaxshi tomonlaridan ilhomlangan va bu ajoyib muqobil.

[Docz](https://github.com/pedronauck/docz) - bu dokumentatsiya saytlarini yaratish uchun Gatsby mavzusi. U hozirda Docusaurusga nisbatan kamroq xususiyatlarga ega.

### Next.js

[Next.js](https://nextjs.org/) yana bir juda mashhur gibrid React freymvorki. U sizga yaxshi dokumentatsiya veb-saytini yaratishga yordam berishi mumkin, ammo u dokumentatsiya ishiga yo'naltirilmagan va Docusaurus tayyor holda taqdim etadigan narsalarni amalga oshirish uchun ancha ko'proq ish talab qiladi.

[Nextra](https://github.com/shuding/nextra) ‚Äî Next.js ustida qurilgan o'ziga xos statik sayt generatori. U hozirda Docusaurusga nisbatan kamroq xususiyatlarga ega.

### VitePress

[VitePress](https://vitepress.dev/) Docusaurus bilan ko'p o'xshashliklarga ega ‚Äì ikkalasi ham kontentga yo'naltirilgan veb-saytlarga juda katta e'tibor qaratadi va tayyor holda maxsus dokumentatsiya funksiyalarini taqdim etadi. Biroq, VitePress Vue, Docusaurus esa React ustiga qurilgan. Agar siz Vue asosidagi yechimni istasangiz, VitePress munosib tanlov bo'ladi.

### MkDocs

[MkDocs](https://www.mkdocs.org/) ‚Äî bu Docusaurusga o'xshash qadriyatlarga ega mashhur Python statik sayt generatori.

Agar sizga bir sahifali ilova (SPA) kerak bo'lmasa va React'dan foydalanishni rejalashtirmasangiz, bu yaxshi tanlov.

[Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) ‚Äî chiroyli mavzu.

### Docsify

[Docsify](https://docsify.js.org/) dokumentatsiya veb-saytini yaratishni osonlashtiradi, lekin u statik-sayt generatori emas va SEO uchun qulay emas.

### GitBook

[GitBook](https://www.gitbook.com/) juda toza dizaynga ega bo'lib, ko'plab ochiq kodli loyihalar tomonidan ishlatilgan. Uning diqqat markazi ochiq kodli vosita bo'lishdan tijorat mahsulotiga o'tishi bilan, uning ko'plab talablari ochiq kodli loyihalarning dokumentatsiya saytlarining ehtiyojlariga mos kelmay qoldi. Natijada, ko'pchilik boshqa mahsulotlarga o'tishga majbur bo'ldi. Reduxning Docusaurusga o'tgani haqida [bu yerda](https://github.com/reduxjs/redux/issues/3161) o'qishingiz mumkin.

Hozirda GitBook faqat ochiq kodli va notijorat jamoalar uchun bepul. Docusaurus esa barcha uchun bepul.

### Jekyll

[Jekyll](https://github.com/jekyll/jekyll) eng yetuk statik sayt generatorlaridan biri bo‚Äòlib, foydalanish uchun ajoyib vosita bo‚Äòlgan ‚Äî aslida, Docusaurusdan oldin Facebookning ko‚Äòpchilik Open Source veb-saytlari Jekyllda yaratilgan! Ishni boshlash juda oddiy. Biz Jekyll bilan statik sayt yaratish kabi dasturchi bilan ishlash qulayligi tajribasini yaratmoqchimiz.

Statik yaratilgan HTML sahifalariga `<script />` teglar orqali interaktivlik qo ªshish usulidan farqli o ªlaroq, Docusaurus saytlari to ªlaqonli React ilovalari hisoblanadi. Zamonaviy JavaScript ekotizimining vositalaridan foydalanib, biz dokumentatsiya saytlarining ishlash samaradorligi, resurslarni shakllantirish jarayoni va optimallashtirish, shuningdek, sozlash qulayligi bo ªyicha yangi standartlarni o ªrnatishga umid qilamiz.

### Rspress

[Rspress](https://rspress.dev/) ‚Äì bu Rust tilida yozilgan Rspack bundleri asosida ishlaydigan tezkor statik sayt generatori. U MDX (ya'ni, React komponentlari bilan ishlovchi Markdown) orqali kontent yaratishni, Integratsiyalashgan matnli qidiruvni, ko'p tilli qo'llab-quvvatlashni (i18n) va plaginlar yordamida kengaytirish imkoniyatini taqdim etadi. Chiroyli dokumentatsiya va statik veb-saytlar yaratish uchun mo'ljallangan Rspress, oson deploy qilinadigan statik HTML fayllarini hosil qiladi.

Rspress va Docusaurus ancha o'xshash. Ularning har ikkisining ham o'ziga xos afzalliklari va kamchiliklari bor. Rspress yaqinda yaratilgan va sayt yaratilishini tezlashtiradigan zamonaviy infratuzilmadan foydalanadi. Docusaurus esa o'zining yetukligi, keng qamrovli imkoniyatlari, moslashuvchanligi va kuchli jamoasi bilan ajralib turadi. U shuningdek, ishlash samaradorligi bo'yicha raqobatbardosh bo'lib qolish uchun o'z infratuzilmasini muntazam ravishda [modernizatsiya qilib boradi](https://github.com/facebook/docusaurus/issues/10556).
